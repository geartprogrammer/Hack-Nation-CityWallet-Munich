Scene 2 of 5
On-device scoring
6 sensors, zero tracking — all intelligence runs on the phone
📍
GPS + Haversine
Calculates distance to every merchant. Filters >2km. Scores by proximity bands.
+40 pts if <100m
🏃
Accelerometer
Detects still / walking / vehicle from motion variance over 20 samples.
+20 pts if standing nearby
☀️
Ambient Light
Reads lux level to detect outdoor vs indoor. Outdoor users are more likely to walk in.
+10 pts if outdoors
🔋
Battery Level
Low battery + far distance = bad time to suggest a walk. Penalizes far offers.
-30 pts if <10% + far
🌦️
Weather Context
Cached weather estimate. Cold day + hot drink offer = high match. Hot day + iced drink.
+15 pts weather match
🕒
Local History
Accept/decline history in localStorage. Boosts liked merchants, suppresses declined ones.
+15 / -20 pts from history
Scene 3 of 5
"Fill my seats" pipeline
One button triggers: weather + user count + AI analysis + push delivery
1
Merchant taps "Fill My Seats"
Single POST to /api/fill with merchant auth token. That's the only human input.
POST /api/fill
2
Fetch real-time weather
Calls Open-Meteo API with merchant's lat/lng. Returns temp, precipitation, wind, UV, weather code.
open-meteo.com
3
Count nearby subscribers
Haversine distance filter across all push subscribers. Calculates who's within walking distance.
Haversine filter
4
AI picks the best menu item
Llama 3.3 70B scores the merchant's menu against weather. Cold + rainy = hot drinks score highest. Hot day = iced drinks.
Groq Llama 3.3 70B
5
AI sets optimal discount
Urgency scoring: rain +8%, cold +5%, dead hours +5%, few users +5%. Capped at merchant's max discount.
Dynamic pricing
6
AI writes the push notification
Groq generates a 12-word personal message. "Sound like a friend, not a brand." Falls back to weather-aware templates.
Groq LLM · 12 words max
7
Push to all nearby users
Web Push via VAPID protocol. Each notification carries the full offer payload: item, price, discount, merchant location, reasoning.
pywebpush + VAPID
Scene 4 of 5
Vision AI + wallet passes
AI-powered onboarding and native pass delivery
📷
Menu Parsing — GPT-4o Vision
Merchant takes a photo of their printed menu. Image is sent as base64 to GPT-4o Vision. The model extracts every item name and price, returning clean structured JSON. Menu populates the merchant profile automatically — zero manual data entry.
POST /api/merchant/parse-menu
{ "image": "base64..." }
→ [{"name":"Cappuccino","price":3.50}, ...]
🌠
AI Offer Reasoning
Every generated offer includes full AI reasoning visible to the merchant: why this item, why this discount %, what signals drove the decision. Complete transparency into the AI's logic.
"Picked Hot Chocolate because it's cold and
rainy. Set 28% off: rain + dead hours =
maximum incentive needed."

Apple Wallet .pkpass
Real .pkpass files: signed ZIP archives containing pass.json, manifest, icons, and a CMS signature. Includes geofence locations — iOS shows the pass on the lock screen when the user is near the cafe. Supports Apple's webServiceURL protocol for live updates.
Content-Type: application/vnd.apple.pkpass
"locations": [{"lat":48.16,"lng":11.58}]
→ Lock-screen notification at ~100m
🤖
Google Wallet JWT
Google Wallet passes are encoded as JWTs containing the full offer object. User taps "Save to Google Wallet" — no file download needed. Android surfaces the pass when the user is near the merchant's coordinates.
JWT { "genericObjects": [{ ... }] }
→ pay.google.com/gp/v/save/{jwt}
→ Android lock-screen at geofence
